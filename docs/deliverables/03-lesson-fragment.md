# Урок: «Мини-MVP за 2 часа» — встраиваем AI-фичу в сервис записи

## Зачем вам этот навык

За последние два года стоимость «попробовать гипотезу» упала в десять раз. Раньше, чтобы проверить идею вроде «сервис, где клиент перед сессией с коучем отвечает на пять вопросов, а AI готовит для коуча гипотезу», нужны были фронтенд-разработчик, бэкенд-разработчик, дизайнер и две недели. Сегодня — один человек и один вечер, если он умеет разговаривать с AI как с коллегой.

Этот навык важнее, чем умение писать код руками. Когда вы — продакт или предприниматель, ваша сила не в том, чтобы сделать «правильную архитектуру», а в том, чтобы за вечер собрать что-то рабочее, показать пяти людям и понять, что менять. Дальше у нас будут проекты сложнее, но базовая привычка — «идея → MVP за вечер» — закладывается на этом модуле.

Кейс, который мы разберём: сервис записи к коучу личностного роста со встроенной AI-диагностикой. Клиент проходит запись → отвечает на пять вопросов внутри формы → AI генерирует короткий документ для клиента и коуча. У нас уже есть рабочая версия: https://66dplus.github.io/mindmatch/ — посмотрите, пощёлкайте, потом возвращайтесь.

## Что вы сделаете за 2 часа

**Шаг 1. Опишите продукт в 4 строки** (10 мин).
В одном файле — JTBD («что хочет пользователь»), 3 ключевые фичи MVP, 1 метрика успеха, и список «что НЕ входит». Последнее — самое важное. Если у вас в «не входит» меньше пяти пунктов, значит, скоуп раздут. Урезайте.

**Шаг 2. Сгенерируйте интерфейс через AI** (30 мин).
Откройте Lovable (или, если без VPN — Qwen Coder Agent на chat.qwen.ai). Опишите экран **с тоном и атмосферой**, а не с компонентами. Не «форма с тремя полями» — а «как бережный литературный журнал, тёплая палитра, серифный шрифт». AI выбирает лучше, чем мы, если задана эмоциональная рамка.

**Шаг 3. Подключите базу данных** (20 мин).
Заведите бесплатный аккаунт на Supabase (без SMS, через email или GitHub). Создайте таблицы: `services`, `available_slots`, `bookings`. Скопируйте URL проекта и `anon key` в `.env.local` вашего фронта. Никакого SQL руками — попросите AI сгенерировать миграцию по описанию схемы.

**Шаг 4. Добавьте AI-фичу через Edge Function** (30 мин).
Это магия: ваше приложение не вызывает OpenAI / Claude напрямую. Оно зовёт **свою** функцию в Supabase, а она уже идёт к LLM. Зачем? API-ключ остаётся на сервере — иначе любой посетитель сайта мог бы его украсть. Edge-функция — это маленький скрипт на TypeScript, который Supabase запускает «в облаке», без сервера.

**Шаг 5. Задеплойте на GitHub Pages** (20 мин).
Vercel и Netlify в 2026 требуют SMS-верификацию. Из России без VPN это невозможно. Зато GitHub Pages — бесплатно, без верификации, и сразу даёт публичную ссылку вида `username.github.io/project`. Деплой автоматизируется через GitHub Actions: push в `main` → сайт обновился.

**Шаг 6. Покажите пяти людям** (10 мин).
Откройте ссылку на телефоне, дайте пятерым из вашего ближнего круга. Запишите три вещи: где они спотыкались, что им непонятно, что они хвалили. Это ваш первый продакт-аналитика.

## Альтернативные стеки

**Базовый стек (если есть VPN)**: Claude Code / Cursor для написания кода, Lovable / v0 для генерации UI, Supabase для БД, OpenRouter для AI-фичи, GitHub Pages для деплоя.

**Стек без VPN (для студентов из РФ)**: **Qwen Coder Agent** (chat.qwen.ai, бесплатно, без VPN) для генерации кода + UI, Supabase (работает в РФ), **GigaChat API** от Сбера для AI-фичи (оплата рублями), GitHub Pages для деплоя.

**Стек «совсем без денег»**: Qwen Coder Agent, **локальная SQLite-БД** в браузере через библиотеку `sql.js` (вообще без бэкенда), GigaChat бесплатные 1000 токенов/мес, GitHub Pages. Минус: ваши данные не сохранятся между устройствами клиента, годится только для самой первой проверки идеи.

## Типичная ошибка: CORS-блокировка из браузера

Самая частая боль на этом этапе: фронт деплоится, AI-функция деплоится, но на странице ничего не происходит. В консоли браузера красным: `blocked by CORS policy`.

Что произошло: ваша страница живёт на `github.io`, а функция — на `supabase.co`. Браузер не разрешает «общаться через границу домена» без явного разрешения от сервера. Это защита от подмены: иначе любой сайт мог бы дёргать любой API под вашим именем.

**Как починить через AI**: скопируйте текст ошибки целиком и отправьте промптом:

> «У меня ошибка CORS: [вставьте текст]. Вот код моей edge-функции: [вставьте код]. Что добавить, чтобы заработало?»

AI сразу узнает паттерн и предложит добавить в каждый ответ функции заголовки `Access-Control-Allow-Origin: *` и обработать preflight-запрос (метод OPTIONS — браузер шлёт его перед основным POST, спрашивая «можно ли»). Скопируйте предложенный код, замените свой, перезадеплойте функцию. 90% случаев CORS лечатся за один промпт.
